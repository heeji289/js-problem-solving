## 1. 문제 설명

사용자 이름과 추천인 이름이 주어진다.
사용자는 추천인에게 수익금의 10%를 떼준다.
그랬을 때 사용자가 얻은 수익금의 합을 구해라.

## 2. 내 접근 방식

1. enroll과 referral을 보고 트리를 만든다 (Node 형태로)
2. seller랑 amount 순회하면서 수익금 Map을 만든다 (\*100원)
3. 3에서 만든 Map을 기준으로 이익금을 분배해 계산하도록 한다
   1. 수익금 1200 스타트
      1. 내 거에 1080 기록, 부모한테 120 넘김 …
   2. 부모가 없을 대까지 & 금액의 10%가 1원 미만일 때까지 넘김
   3. 계산해서 수익 Map 저장
4. enroll 순회하면서 수익 Map에서 값을 꺼낸다.

## 3. 틀린 이유

일단 내 접근 방식에서 트리를 노드 형태로 만들어야 하는데 이 부분에서 막힘 (Tree를 어떻게 표현할지? find, add 메소드 구현)
→ 이렇게 풀어보다가 이거 아닌 것 같다 생각이 들어 책 답지를 봤음

책 보고 어느 정도 아이디어 얻어서 내 접근 방식에서 3,4 부분을 해보려고 시도
→ 이게 parent를 찾아 올라가도록 해야 하는데 어떻게 구현해야 할 지 모르겠어서 포기

## 4. 올바른 접근 및 풀이

1. 먼저 이름과 추천인을 매칭해둔다 (Object의 key로 이름, value로 추천인 이름)
2. enroll 기준으로 Object를 만들고 0으로 초기화해둔다 (수익금을 담을 Object)
3. seller를 순회. 포인터로 현재 이름과 현재 금액을 만들어둔다.
   1. 내부에서 while을 돌리는데 현재 이름이 '-'가 아니고 (부모이면 끝났으므로) 현재 금액이 0 이상일 때까지 순회
   2. 순회하면서 money에서 10% 버림 처리한 값을 빼서 수익금을 계산해주고 referral을 기준으로 해서 현재 이름을 내 추천인으로 결정함
   3. 2를 반복
4. 최종적으로 2에서 만든 수익금 Object를 보고 수익금을 출력하면 된다.
