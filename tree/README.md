# 트리

## 트리의 개념

### 트리 특성을 활용하는 분야

- 자동 완성 기능 → 검색 엔진에서 자동 검색어 추천 기능도 트라이라는 걸 활용한 것
- DB → 트리를 활용해 데이터를 구조화하고 인덱싱 (B-트리, B+트리)

```
- prefix tree라고도 불린다
- 각 노드는 문자 하나를 저장하고 루트에서 특정 노드까지 내려가는 경로가 곧 하나의 단어 혹은 접두사가 됨
- 예를 들어 apple, apply가 있다면 a-p-p-l까지 공용으로 노드를 사용하고 이후에 e, y로 갈라짐
```

### 트리 용어

- **노드**: 트리를 구성하는 요소
  - 노드 중 가장 위에 있는 노드를 **루트 노드**라고 함
  - 자식이 없는 노드를 **리프 노드** (말단 노드)라고 함
- **간선(에지)**: 노드 간 연결 선
- 부모-자식 관계
  - 간선으로 연결된 노드 중 상대적으로 위에 있는 노드를 **부모 노드**, 아래에 있는 것을 **자식 노드**라고 함
  - 같은 부모를 갖는 노드를 **형제 노드**라고 함
- **차수**: 특정 노드에서 아래로 향하는 간선의 개수

## 이진 트리

### 개념

- **이진 트리**: 모든 노드의 최대 차수가 2를 넘지 않는 트리 (간선이 최대 2개인 트리)
  <img src="https://upload.wikimedia.org/wikipedia/commons/thumb/f/f7/Binary_tree.svg/250px-Binary_tree.svg.png">

### 표현 방법

1. 배열로 표현
   - 규칙
     - 루트 노드는 인덱스 1번. 왼쪽 자식 노드는 `부모 인덱스 * 2`, 오른쪽은 `부모 인덱스 * 2 + 1`
     - 인덱스 0부터 시작하면 왼쪽은 `부모 인덱스 * 2 + 1`, 오른쪽은 `부모 인덱스 * 2 + 2`

   - 배열을 이용하면 실제 노드 개수보다 많은 공간을 차지함 (자식이 없거나 쓰지 않는 인덱스는 그냥 비어있어 메모리 낭비가 될 수 있음)
   - 메모리가 넉넉하다면 배열로 구현해도 괜찮다.

2. 포인터로 표현
   <img src="https://encrypted-tbn0.gstatic.com/images?q=tbn:ANd9GcRgZrZG_fnShaJDyvgUunNlrQom6ipunpMfAA&s">

### 순회하기

1. 전위 순회 (preorder): 부모 노드 → 왼쪽 자식 노드 → 오른쪽 자식 노드 순
2. 중위 순회 (inorder): 왼쪽 자식 노드 → 부모 노드 → 오른쪽 자식 노드
3. 후위 순회 (postorder): 왼쪽 자식 노드 → 오른쪽 자식 노드 → 부모 노드

## 이진 탐색 트리

### 개념

- 규칙은 왼쪽 자식 노드의 값이 부모 노드보다 작고 오른쪽 자식 노드의 값이 부모 노드보다 크다. (정렬 규칙을 가진 이진 트리다)

### 탐색하기

1. 타겟이 현재 노드의 값과 같으면 탐색을 종료
2. 만약 타겟이 더 크면 오른쪽, 작으면 왼쪽 노드를 탐색
3. 찾으면 종료, 노드가 없어질 때까지 탐색

### 특징

- 시간 복잡도
  - 트리가 균형이 잡힌 상태일 때는 삽입, 탐색 시 O(logN)
  - 균형이 맞지 않을 때는 배열과 비슷함 (한 쪽으로 치우쳐지는 등)
- 균형 이진 탐색 트리 (balanced binary search tree)
  - AVL 트리, 레드-블랙 트리 등으로 구분하여 부름
  - 이진 트리의 탐색 연산 횟수가 트리 높이에 비례함
